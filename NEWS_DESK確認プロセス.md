# NewsDesk 確認プロセス定式化

**作成日：** 2026-04-17  
**バージョン：** 1.0

---

## 概要

NewsDesk のニュース記事確認を**定式化**し、確認していない日数があった場合も自動的に対応するプロセスです。

### 問題背景

- 手動確認のため、日によって確認し忘れる可能性がある
- 複数日確認していない場合、全期間のデータを効率的に確認したい
- 確認の進捗を自動追跡したい

### 解決方法

**`newsdesk-check.py` スクリプト** で以下を自動化：

1. **前回確認日を記録** → `~/.newsdesk_check_history.json`
2. **期間を自動計算** → 前回確認日から本日までの日数を計算
3. **全期間のデータを一括取得** → Supabase から該当期間の全記事取得
4. **統計表示** → 日別・地域別・配信元別の集計を自動表示
5. **確認日を更新** → スクリプト実行後、新しい確認日を記録

---

## 使用方法

### 基本実行

```bash
python3 newsdesk-check.py
```

### 動作フロー

```
1️⃣ 前回確認日を確認
   ├─ 初回実行 → 過去7日間を確認
   └─ 2回目以降 → 前回確認日から本日までを確認

2️⃣ 期間データを取得
   └─ Supabase articles テーブルから該当期間の記事をすべて取得

3️⃣ 統計を自動集計
   ├─ 📅 日別統計（グラフ表示）
   ├─ 🗺️ 地域別統計（上位表示）
   ├─ 📰 情報種別
   └─ 📡 配信元別（上位10）

4️⃣ 確認日を記録
   └─ ~/.newsdesk_check_history.json に本日の日付を保存
```

## 確認履歴の仕組み

### 履歴ファイル場所

```
~/.newsdesk_check_history.json
```

### ファイル内容

```json
{
  "last_check_date": "2026-04-17"
}
```

### リセット方法

```bash
# 履歴をリセット（初期化）
rm ~/.newsdesk_check_history.json

# 次回実行時に過去7日間で再初期化
python3 newsdesk-check.py
```

## 実装詳細

### 前回確認日から本日までの計算

```python
if last_check:
    # 前回確認日と本日の差分を計算
    days_since = (datetime.fromisoformat(today) - datetime.fromisoformat(last_check)).days
    date_from = last_check
else:
    # 初回実行時は過去7日間
    date_from = (datetime.fromisoformat(today) - timedelta(days=7)).date().isoformat()
    days_since = 7
```

### Supabase クエリ

```sql
SELECT * FROM articles 
WHERE published_at >= '{date_from}' 
  AND published_at < '{date_to_next}'
ORDER BY published_at DESC
LIMIT 1000
```

## よくある質問

### Q1: 複数日確認していない場合はどうなる？

**A:** 自動的に全期間のデータが取得され、統計が表示されます。

例：
```
金曜日に実行 → 金曜分を確認
月曜日に実行 → 金曜日〜月曜日（4日分）のデータを表示
```

### Q2: 確認履歴をリセットしたい

**A:** 履歴ファイルを削除

```bash
rm ~/.newsdesk_check_history.json
```

次回実行時に過去7日間で初期化されます。

### Q3: 特定期間だけ確認したい

**A:** スクリプトを修正して直接クエリを実行：

```python
# 特定期間をハードコード
articles = get_articles('2026-04-10', '2026-04-17')
```
