## 2. 実行フロー

### 全体処理順序

```
[8:30 AM] タスク開始
    ↓
[Step 1] /newsdesk-fetch スキル実行（Gmail メール取得 + 記事抽出）
    ↓
[Step 2] newsdesk-fetch.py 実行（Supabase 登録）
    ↓
[Step 3] newsdesk-check.py 実行（複数日確認 + 統計表示）
    ↓
[完了] 実行結果ログ & 通知
```

### Step 1: メール取得＆記事抽出
**スキル:** `/newsdesk-fetch`

動作：
- Gmail MCP 経由で昨日～本日のメールを自動取得
- Claude Haiku で記事を AI 抽出
- 対応ソース：
  - iJAMP（行政情報）
  - Google アラート
  - 広島県地域力創造課
  - 日本農業新聞
- 出力形式：JSON（記事オブジェクト）

### Step 2: 記事登録
**スクリプト:** `newsdesk-fetch.py`

動作：
```bash
cd "/Users/ryotaz05/Library/CloudStorage/Dropbox/デスクトップ/0_知識/Claude/NewsDesk"
python3 newsdesk-fetch.py
```

機能：
- Step 1 の JSON を stdin で受け取る
- URL ベース重複チェック（external_id）
- Supabase REST API で articles テーブルに INSERT
- 登録日時を `~/.newsdesk_fetch_history.json` に記録

出力例：
```
[08:32:15] ✓ 登録: 12 件
[08:32:15] ⚠ 重複: 2 件
[08:32:15] ✗ エラー: 1 件
```

### Step 3: 複数日確認＆統計
**スクリプト:** `newsdesk-check.py`

動作：
```bash
cd "/Users/ryotaz05/Library/CloudStorage/Dropbox/デスクトップ/0_知識/Claude/NewsDesk"
python3 newsdesk-check.py
```

機能：
- 前回確認日から本日までのデータを自動取得
- 日別・地域別・配信元別の統計を計算
- 確認日を `~/.newsdesk_check_history.json` に記録

出力例：

## 4. エラーハンドリング

### 監視対象エラー

| エラー内容 | 検出方法 | ログメッセージ | 対応 |
|---|---|---|---|
| **Gmail 接続エラー** | /newsdesk-fetch が空 JSON 返却 | `⚠ 指定期間にメールが見つかりません` | スキップ、次回自動リトライ |
| **記事抽出失敗** | JSON パース失敗 | `✗ JSON パース失敗: [詳細]` | エラーログ記録、通知 |
| **DB 接続エラー** | Supabase API 4xx/5xx | `✗ データベースエラー: [HTTP ステータス]` | 部分成功で続行 |
| **スクリプト実行失敗** | Python プロセス非ゼロ終了 | `✗ [スクリプト名] 実行失敗` | 詳細スタックトレース記録 |

### エラー発生時の動作

1. **軽度（メールなし）** → スキップして続行、ログ記録
2. **中度（一部登録失敗）** → 成功分を記録、失敗分をエラーログ
3. **重度（スクリプト完全失敗）** → 通知でユーザーに報告

## 9. Cron 式の詳細

### `30 8 * * *` の意味

```
分      時間   日付  月    曜日
30      8     *     *     *
↓       ↓     ↓     ↓     ↓
毎日 08:30 実行（祝日も含む）
```

| フィールド | 値 | 意味 |
|---|---|---|
| 分 | 30 | 30分（:30） |
| 時間 | 8 | 08時（朝 8時） |
| 日付 | * | 毎日 |
| 月 | * | 毎月 |
| 曜日 | * | 毎曜日 |

### カスタマイズ例

```bash
# 毎日 7:00 AM
0 7 * * *

# 平日 9:00 AM（月～金）
0 9 * * 1-5

# 毎週月曜 8:30 AM
30 8 * * 1

# 毎月 1日 8:30 AM
30 8 1 * *
```

## 11. 既知の制限事項

### Gmail メール取得

- **キャパシティ:** 昨日～本日のメールのみ
- **リトライ:** 差分取得なので、同じメールは再処理されない
- **上限:** 1000件/実行

### 記事登録

- **重複チェック:** URL ベース（完全一致のみ）
- **タイムアウト:** Supabase REST API の制限に依存

### 確認チェック

- **統計精度:** Supabase クエリ結果に依存
- **表示形式:** ターミナル固定（HTMLやJSONは出力しない）
