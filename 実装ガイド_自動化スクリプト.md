## 概要

メール取り込み処理を以下の2つのコンポーネントに分割しました：

| コンポーネント | 役割 | 実装 | トークン削減 |
|---|---|---|---|
| **スキル `/newsdesk-fetch`** | Gmail MCP でメール取得 → 記事抽出・AI解析 | Claude Code スキル | Haiku 使用（1/3削減） |
| **スクリプト `newsdesk-fetch.py`** | JSON受け取り → Supabase に INSERT | Python スクリプト | SQL直接実行（キャッシュ活用） |

## 使用方法

### **方法B：スクリプト経由での登録**

```bash
EOF
```

**フロー図：**
```
Claude Code
  ↓

  (Gmail MCP + Haiku)

  ↓
articles テーブルに登録完了
```

## スクリプト内部の処理

### `newsdesk-fetch.py` の流れ

```python
1. read_articles_from_stdin()
   → スキルからの JSON を受け取る

2. check_duplicates()
   → Supabase で existing external_id をチェック

3. register_articles()
   → 新規レコードのみ INSERT
   → タグリンク（article_tags）も同時登録

4. log() で進行状況を表示
   → ✓登録: N件 / ⚠重複: M件 / ✗エラー: K件
```

### トークン削減の工夫

| 項目 | 方法 | 効果 |
|---|---|---|
| **モデル選択** | Haiku（4M tokens/分） | 約1/3削減 |
| **バッチ処理** | 5ソース同時処理 | API呼び出し1回 |
| **構造化出力** | JSON固定フォーマット | プロンプト最小化 |
| **キャッシュ** | Supabase 重複チェック | 不要なINSERT排除 |
| **スクリプト処理** | Python 直接実行 | API呼び出し不要 |

→ **推定削減：80-90%**（前セッション比）

## トラブルシューティング

### Q1: スキル実行時に「Gmail MCP エラー」が出る

**A:** Claude Desktop の設定を確認

```bash
# Claude Desktop が起動しているか確認
ps aux | grep Claude

# 起動していなければ起動
open -a "Claude"
```

### Q2: JSON がパースできないエラーが出る

**A:** 出力JSON形式を確認

```json
// 正しい形式
{
  "articles": [
    {
      "source": "47",
      "title": "...",
      "url": "...",
      "snippet": "...",
      "date": "2026-04-15",
      "category": "...",
      "tags": ["...", "..."],
      "original_source": "..."
    }
  ],
  "summary": "...",
  "count": 1
}
```

### Q3: 記事が抽出されない

**A:** Gmail にメールが到着しているか確認

1. Gmail で直接確認
2. 日付範囲が正しいか確認（自動計算：昨日～今日）
3. メール本文に記事リンクが含まれているか確認

### Q4: Supabase 登録時にエラーが出る

**A:** API キーを確認

```bash
# スクリプトの SB_URL, SB_KEY が正しいか確認
grep "SB_URL\|SB_KEY" newsdesk-fetch.py
```
