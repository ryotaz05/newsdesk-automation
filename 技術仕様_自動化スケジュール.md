## 1. スケジュール仕様

### Cron 式

```
30 8 * * *
```

**構造:**
```
分      時間   日付  月    曜日
│       │      │     │     │
│       │      │     │     └─ 曜日 (0-6, 0=日)  → * (全曜日)
│       │      │     └────── 月 (1-12)          → * (全月)
│       │      └──────────── 日 (1-31)          → * (毎日)
│       └─────────────────── 時 (0-23)          → 8 (午前8時)
└────────────────────────── 分 (0-59)          → 30 (30分)
```

### 実行パターン

| 日時 | Cron | 実行 |
|---|---|---|
| 毎日 8:30 | `30 8 * * *` | ✅ |
| 特定時刻（日替わり）| `30 8 * * *` | ✅ |
| 祝日・休日 | `30 8 * * *` | ✅（祝日も実行） |

### タイムゾーン

- **設定:** ユーザーのローカルタイムゾーン（自動判定）
- **例:** JST (UTC+9) での場合、08:30 JST = 23:30 UTC (前日)
- **確認方法:**
  ```bash
  date +%Z  # JST が表示される
  TZ='' date  # UTC で表示
  ```

## 4. 入出力仕様

### Step 1: /newsdesk-fetch (入力)

**入力:** なし（Gmail MCP で自動取得）

**内部動作:**
- Gmail MCP 経由で以下をクエリ：
  ```
  from: (iJAMP|google-alert|広島県|農業新聞)
  after: [昨日日付]
  before: [本日日付+1]
  ```
- Claude Haiku で各メール本文から記事を抽出
- JSON 配列として stdout に出力

**出力フォーマット (JSON):**
```json
[
  {
    "title": "記事タイトル",
    "source_id": "iJAMP",
    "region": "広島県",
    "published_at": "2026-04-20T08:00:00",
    "original_source": "新聞社名",
    "url": "https://...",
    "info_type": "ニュース",
    "tags": ["重要", "自治体"]
  },
  ...
]
```

### Step 2: newsdesk-fetch.py (処理)

**入力:** Step 1 の JSON (stdin)

**処理アルゴリズム:**

```python
1. JSON を stdin から読み込む
2. 各記事について以下を実行：
   a. external_id を生成（URL を Base64 エンコード）
   b. Supabase から external_id で検索（重複チェック）
   c. 重複なし → INSERT、重複あり → スキップ
3. 実行結果を集計：
   - 登録件数
   - 重複件数
   - エラー件数
4. ~/.newsdesk_fetch_history.json を今日の日付で更新
5. 結果を stdout に出力
```

**出力フォーマット:**
```
[08:32:15] ✓ 登録: 12 件
[08:32:15] ⚠ 重複: 2 件
[08:32:15] ✗ エラー: 1 件
```

### Step 3: newsdesk-check.py (処理)

**入力:** なし（Supabase REST API で自動取得）

**データ取得クエリ:**

```sql
SELECT 
  id, title, source_id, region, published_at, 
  original_source, created_at, info_type
FROM articles
WHERE published_at BETWEEN '2026-04-19' AND '2026-04-20T23:59:59'
ORDER BY published_at DESC
LIMIT 1000
```

**出力フォーマット:**
```

  2026-04-19: ████████ 8 件
  2026-04-20: ██████████ 10 件

🗺️ 地域別統計

  東京都: 4 件 (20.0%)

📰 情報種別
  ニュース: 15 件
  通知: 3 件

📡 配信元別（上位10）
  朝日新聞: 4 件
  読売新聞: 3 件
  共同通信: 2 件

📊 確認サマリー
  期間: 2026-04-19 ～ 2026-04-20
  日数: 2 日
```

## 8. エラーコード仕様

### Step 1 エラー

| コード | 内容 | ログメッセージ | 対応 |
|---|---|---|---|
| E1001 | Gmail 接続失敗 | `✗ Gmail MCP 接続エラー` | リトライ、次回実行で再試行 |
| E1002 | メール取得ゼロ | `⚠ メール取得なし` | 続行（データなし）|
| E1003 | JSON パース失敗 | `✗ JSON パース失敗: [詳細]` | 停止、通知 |

### Step 2 エラー

| コード | 内容 | ログメッセージ | 対応 |
|---|---|---|---|
| E2001 | Supabase 接続失敗 | `✗ DB 接続エラー` | リトライ |
| E2002 | INSERT 失敗 | `✗ INSERT 失敗: [詳細]` | 該当記事スキップ、続行 |
| E2003 | URL decode 失敗 | `⚠ external_id 生成失敗` | デフォルト値使用 |
| E2004 | 履歴ファイル更新失敗 | `⚠ 履歴ファイル保存失敗` | ログ記録、続行 |

### Step 3 エラー

| コード | 内容 | ログメッセージ | 対応 |
|---|---|---|---|
| E3001 | API クエリ失敗 | `✗ Supabase クエリエラー` | リトライ |
| E3002 | データなし | `ⓘ 確認期間にデータなし` | 続行（正常） |
| E3003 | 統計計算エラー | `✗ 統計計算エラー` | デフォルト表示 |

## 9. セキュリティ仕様

### API キー管理

| キー | 種類 | 権限 | 保管場所 |
|---|---|---|---|
| `sb_publishable_8u...` | Publishable Key | SELECT, INSERT | hardcoded |
| *（なし）* | Secret Key | DELETE, UPDATE | 不使用 |

**セキュリティ考慮:**
- Publishable Key のみ使用（読み書き制限）
- Secret Key は不使用（削除不可）
- API Key は環境変数で管理推奨（現在 hardcoded）

### データ保護

| データ | 保護方法 | 備考 |
|---|---|---|
| 記事 URL | external_id（Base64） | 重複検出用のみ |
| メール内容 | AI 抽出のみ保存 | 原メール保存なし |
| 個人情報 | スキップ | 記事メタデータのみ |

### C. FAQ

**Q: 8:30 ちょうどに実行されない**

A: Cron には数分のジッター（遅延）が適用されます。実際の実行時刻は 8:30～8:40 の間です。

**Q: 前日のメールが取得されない**

A: Gmail フィルター設定を確認してください。iJAMP など送信者でフィルターしている場合、メール受信設定を見直してください。

**Q: 記事が重複検出される**

A: URL が完全に一致する場合のみ重複と判定されます。クエリパラメータが異なる場合は別記事として登録されます。

**Q: 統計表示が文字化けしている**

A: ターミナルの文字エンコードが UTF-8 になっているか確認してください。

**Q: 自動実行が止まった**

A: Claude Desktop が起動しているか確認してください。Claude Desktop を再起動すると自動再開されます。
